  id, name, destination, start_date, end_date,

  status: "planning" | "booked" | "completed",
  flights: [], hotels: [], car_rental: null,

  expense_type: "aws" | "personal" | "mixed",
  notes: "",
  created_at, updated_at

3. FLIGHT WATCH
Store price watches in Replit Database under key: flight-watch:{uuid}
Schema:
{
  id, route_from, route_to, 
  date_range: { earliest, latest },
  cabin_class: "economy" | "business" | "first",
  price_threshold_usd: number,
  airline_preference: "Delta",
  status: "active" | "triggered" | "expired",
  created_at, last_checked, triggered_at
}

Daily at 8:00 AM UTC: check all active flight watches
via web search, update prices, trigger alerts via 
sendSystemAlert if price drops below threshold.

6. REQUEST HANDLING
The main POST /agents/travel-agent endpoint receives natural 
language requests and handles:

"plan a trip to [destination]" 
  → Create trip record, search flights from BDL/JFK/LGA on Delta,
    search Hilton/Marriott hotels, build itinerary, return plan

"set up a flight watch for [route] under $[amount] [cabin]"
  → Create flight-watch record, confirm setup

"what are my loyalty balances"
  → Return current loyalty snapshot or prompt to update

"update my [program] balance to [amount]"
  → Update loyalty snapshot

"add travel doc [type] expiring [date]"
  → Store travel document

"help me plan my Italy trip"

  → Search Hilton Diamond and Marriott Platinum properties
    within 1 mile of venue, include transportation options

All responses must include: { agent, output, success, action_taken }
